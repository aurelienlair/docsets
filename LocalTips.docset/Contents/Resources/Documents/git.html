     Command within vim:
     aurelien@linux:~$ :Gpull

# Glog will show the commits of the current file with fugitive 

     Command within vim:
     aurelien@linux:~$ :Glog

     Command within vim:
     aurelien@linux:~$ :Gread

# Gremove will remove the file from the repository and reload the buffer with fugitive 

     Command within vim:
     aurelien@linux:~$ :Gremove
   
     Is the same thing like in command line:
     aurelien@linux:~$ git rm filename.php

# Gmove will rename the current file from the repository with a new name and reload the buffer with fugitive 

     Command within vim:
     aurelien@linux:~$ :Gmove newname.php
   
     Is the same thing like in command line:
     aurelien@linux:~$ git mv filename.php newname.php

# Gdiff will give the diff of local files of the current repository with the remote branch with fugitive 

     Command within vim :
     aurelien@linux:~$ :Gdiff HEAD 
   
     It will show the differences between the local files and the last committed 
     modification for the current branch
   
     Is the same thing like in command line:
     aurelien@linux:~$ git diff HEAD
     
     Same thing with the penultimate version:
     aurelien@linux:~$ :Gdiff HEAD~1

# Greset will reset current HEAD to the specified state with fugitive 

     Command within vim to the penultimate version:
     aurelien@linux:~$ :Greset HEAD~1
   
     Is the same thing like in command line:
     aurelien@linux:~$ git reset HEAD~1

# Gcommit will record the current changes to the repository with fugitive 

     Command within vim:
     aurelien@linux:~$ :Gcommit
   
     Is the same thing like in command line:
     aurelien@linux:~$ git commit

At the top is some diff info about the current position in the file, followed by the actual diff of the source code (called “hunk”), and below that are your available options. Pressing ? will give you a short explanation for each one:

  • y - Yes, add this hunk
  • n - No, don’t add this hunk
  • d - No, don’t add this hunk and all other remaining hunks. Useful if you’ve already added what you want to, and want to skip over the rest
  • s - Split the hunk into smaller hunks. This only works if there’s unchanged lines between the changes in the displayed hunk, so this wouldn’t have any effect in the example above
  • e - Manually edit the hunk. This is probably the most powerful option. As promised, it will open the hunk in a text editor and you can edit it to your hearts content
